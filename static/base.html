Introduction

This document outlines the API endpoints and functionality of a product catalog backend built with Node.js, Express, and MongoDB. The API provides CRUD (Create, Read, Update, Delete) operations for managing products. While it's a best practice, this API doesn't have an authorization protocol. However, rest assured, future versions may include one.

This is the current version of this challenge, created after compiling everything I learned from books, videos, and articles. I have also followed most of Santiago's code, only making changes when necessary to complete the tasks.

I am using S.O.L.I.D as my method, and you can see that when you check the files in the folder controllers. Each file represents one request from our application, with searchCtrl.js having multiple (but separated) requests (all, name, ID, category, brand). I also decided against creating a separate file just for getAllProducts, as I was trying to keep most of the GET requests together.

Running Locally

Create an .env File

The .env file holds the important variables for the whole application which include the database URL, database port, application port, etc.

HOST='localhost'
PORT='3180' # PORT 5000 was already in use on my machine.
MONGO_URL='localhost'
MONGO_PORT=27017
MONGO_DBNAME='product-catalog'

Base URL

Base URL for all API endpoints:

http://localhost:3110

API Endpoints

  • GET /basePage
  • GET /
  • GET /search/name/:searchQuery
  • GET /search/id/:searchQuery
  • GET /search/brand/:searchQuery
  • GET /search/category/:searchQuery
  • PUT /update/:productId
  • DELETE /delete/:productId
  • POST /products

Please be aware that you must use the unique ID created by MongoDB to update and delete a product.

Include the param when searching, updating, or deleting a product and finish your request. Like:

http://localhost:3110/search/name/HP1

HP1, in this case, is a product with the name HP1.

Testing the Application

To test this application, run the following command:

npm test

The program will test all nine methods (base page, insert, search by name, _id, category, brand name, as well as update and delete) in about one second.

To run these tests, I created a function to generate a random number for the 'TestProduct—'. I used ".date" and Math.random to generate random numbers between 0 and 9999 and keep the product names as unique as possible.
